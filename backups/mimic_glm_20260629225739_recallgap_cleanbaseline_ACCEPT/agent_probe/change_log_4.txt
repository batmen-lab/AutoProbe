Step 4 (change):
  - File: train.py
    * Added _group_tpr_dispersion() helper: differentiable surrogate of EOD.
      Uses a soft positive prediction sigmoid((p - 0.5) / 0.05) so the
      penalty has gradients on positive-sample logits and pushes each
      group's positives above the shared threshold. Returns max-min per-
      group TPR gap; groups with no positives are skipped. Done on GPU,
      batch-local, so it integrates with the existing train loop.
    * Extended train_one_epoch() to take eth_dim, fairness_weight, threshold
      kwargs, compute the BCE loss plus fairness_weight * eod_surrogate, and
      backprop the combined loss. eth one-hot is now read from the batch
      dict (already provided by MIMICMortalityDataset.__getitem__).
    * main() now calls train_one_epoch with eth_dim=len(ETH_NAMES),
      fairness_weight=0.5, threshold=0.5. The threshold 0.5 matches the
      shared decision boundary the probe fixes at epoch 1 (epoch-1 grid
      picked t=0.54 in iter 3, close to 0.5). A moderate fairness_weight=0.5
      keeps the BCE signal primary while pushing TPR parity.
  - File: dataset.py — NOT modified; the 'eth' key in the sample dict
    (already added in iter 3) is sufficient for the regularizer.

Expected effect: each batch's loss now directly rewards pulling every
group's positive score above the shared threshold, shrinking per-group TPR
dispersion. Combined with the balanced sampler (which ensures each batch
contains positives from every group), the regularizer should drive EOD
below the 0.05 standard threshold. The dominant residual failure mode is
if the soft-threshold temperature is too low (gradient vanishes for
positives far from 0.5) — in that case the next iteration would tune
temperature up or use a hinge-style fairness penalty, but those are
speculative relative to this single high-leverage change.
