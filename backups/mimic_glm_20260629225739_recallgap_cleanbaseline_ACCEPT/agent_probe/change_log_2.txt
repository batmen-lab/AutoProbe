Step 1–3 (diagnosis):
  - Metric: equal_opportunity_difference = max per-ethnicity recall - min per-
    ethnicity recall, evaluated at a shared threshold fixed on epoch 1's
    validation predictions (here t=0.52).
  - From logs/fairness_eod.csv the per-group recall pattern across all 20
    epochs is dominated by the "other" bucket at recall 0.86-0.90 (max) and
    "black"/"hispanic" at recall 0.46-0.55 (min), giving EOD ~0.35-0.43.
    White and asian sit in the middle (0.70-0.79). The gap is stable from
    epoch 6 onward — the model has already saturated the per-group score shift
    available via its linear weight on the eth one-hot.
  - Root cause: MIMICMortalityDataset concatenates the eth one-hot onto the
    TF-IDF feature vector (dataset.py:42). LogisticRegression is a single
    linear layer, so it can trivially learn a per-group bias term from the
    eth one-hot. The model uses that bias to push "other" past the threshold
    while keeping "black"/"hispanic" below it, inflating recall dispersion
    and hence EOD at the shared threshold. Removing eth from the model's
    input removes this direct lever.

Step 4 (change):
  - File: train.py (around the dataset construction in main()).
  - Change: instantiate both train_ds and val_ds with use_eth=False. The
    MIMICMortalityDataset class already supports this flag and still keeps
    self.eth on the dataset object, which is all the prober needs for
    bucketization (prober._collect reads loader.dataset.eth, not the
    feature vector). A short comment was added explaining the rationale so
    a future reader understands why the model input was decoupled from the
    fairness probe's bucketization signal.

Expected effect: without a direct eth-weighted logit term, per-group score
distributions become much more similar at the shared threshold, so per-group
recall dispersion should drop materially. tail_mean should move well below
0.3775; the dominant failure mode is if the TF-IDF vocabulary itself encodes
group-correlated terms, in which case the gap narrows but does not vanish. In
that case, the next iteration would address it via a group-balanced sampler
or fairness regularizer, but those are speculative relative to this single
high-leverage change.
