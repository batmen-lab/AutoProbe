After:
    parser.add_argument("--batch-balance-strength", type=float, default=1.0)

Rationale
---------
Forces sample_cells() to draw a uniform 0.5/0.5 batch composition (max 8000 cells, ~4000 per batch) instead of the observed-proportion blend (~51/49). With batch_balance_strength=1.0 the encoder is trained against the same 50/50 target that kBET's null hypothesis uses (a cell's neighborhood batch composition matches the global composition), removing the ceiling that left the round-7 metric plateaued at tail_mean 0.744. ELBO anchor is unaffected — balanced subsampling does not change per-cell reconstruction difficulty.

Touched only train.py, only the one default value. Did not modify prober.py, _ProbeCallback, user_analyze (stub `pass`), record()/conclude() call sites, ANCHOR markers, the SCVI constructor / plan_kwargs, or any other hyperparameter.
