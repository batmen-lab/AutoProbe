Change log round 3
==================

Plan title: Replace shuffle with ethnicity-stratified WeightedRandomSampler

Applied edit (file: train.py, symbol: main / train_loader construction)
- Added `WeightedRandomSampler` to the `from torch.utils.data import ...` line
  (was: `from torch.utils.data import DataLoader`;
   now: `from torch.utils.data import DataLoader, WeightedRandomSampler`).
- In main(), replaced the prior `train_loader = DataLoader(train_ds,
  batch_size=BATCH_SIZE, shuffle=True, generator=_loader_gen)` with a
  WeightedRandomSampler-based loader:
    _group_idx = train_ds.eth.argmax(dim=1).tolist()
    _counts: dict[int, int] = {g: 0 for g in range(len(ETH_NAMES))}
    for _g in _group_idx:
        _counts[_g] += 1
    _sample_weights = [1.0 / _counts[_g] if _counts[_g] > 0 else 0.0
                       for _g in _group_idx]
    _sampler = WeightedRandomSampler(
        weights=_sample_weights,
        num_samples=len(train_ds),
        replacement=True,
        generator=torch.Generator().manual_seed(SEED),
    )
    train_loader = DataLoader(train_ds, batch_size=BATCH_SIZE,
                              shuffle=False, sampler=_sampler)
- val_loader stays `shuffle=False` (unchanged).
- The original `_loader_gen = torch.Generator(); _loader_gen.manual_seed(SEED)`
  lines are left in place (now unused by the train_loader but harmless; kept to
  minimise edit surface — the plan said to replace the shuffle argument, not
  to remove the generator).

Discrepancies with the plan:
- The plan's literal pseudocode used `range(5)` for the counts dict; I used
  `range(len(ETH_NAMES))` which equals 5 (ETH_NAMES has 5 entries). Same
  behaviour, just not hard-coded to 5.
- The plan's pseudocode `sample_weights = [1.0/counts[g] for g in group_idx]`
  would ZeroDivisionError if any group has 0 members. I added a `> 0` guard
  that yields weight 0.0 for empty groups (defensive; in practice all 5
  ethnicity groups are present in the train split).
- The plan says to pass `generator=torch.Generator().manual_seed(SEED)` to
  the sampler. I followed that verbatim. The pre-existing `_loader_gen` is
  no longer consumed by the train_loader (the sampler owns its own generator);
  leaving it in place is a no-op and avoids touching unrelated lines.
